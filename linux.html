Linux File Permissions

Improving Security Posture

Overview

This project focuses on updating file permissions for certain files and directories within the projects directory to reflect the appropriate level of authorization, enhancing system security.

Key Tasks

  • Check file and directory details
  • Describe the permissions string
  • Change file permissions
  • Change file permissions on a hidden file
  • Change directory permissions

Technologies Used

  • Linux command-line interface
  • chmod command
  • ls command

Project Details

Checking File and Directory Details

Used the ls -la command to display a detailed listing of file contents, including hidden files.

Understanding Permissions String

Explained the 10-character string representing file permissions, including file type and read, write, and execute permissions for user, group, and other.

Changing File Permissions

Used the chmod command to modify permissions on various files, including removing write access for 'other' users.

Handling Hidden Files

Modified permissions on a hidden file (.project_x.txt) to remove write access and adjust read permissions.

Directory Permissions

Adjusted permissions on the 'drafts' directory to restrict access to only the researcher2 user.

Summary

This project demonstrated the process of checking and modifying file and directory permissions in a Linux environment to match the organization's desired level of authorization. It involved using commands like ls -la to check existing permissions and chmod to make necessary changes, enhancing the overall security posture of the projects directory.
